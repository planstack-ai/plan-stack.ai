Practical Workflow

Theory is great. But how do you actually use this day-to-day? Here's a concrete workflow that puts everything together.

Step-by-Step Workflow

Define a clear, scoped task

Before typing anything, know exactly what you want. "Fix the login bug" is vague. "Fix the null pointer exception in authenticate() when email is empty" is scoped.

Gather minimal context

Identify what Claude absolutely needs. For a bug fix: the broken function, the error message, and immediate dependencies. Nothing more.

Structure your prompt

Put the task description first. Then the essential code. End with any constraints or requirements. Remember: start and end get the most attention.

Review and iterate

If results are incomplete, add context incrementally. If they're wrong, clarify the task. Don't just paste more code and hope for the best.

CLAUDE.md Setup

Create a CLAUDE.md file in your project root. This file is automatically read at the start of every conversation, providing persistent context.

Include:

  • Project-specific conventions (naming, patterns)
  • Tech stack details that affect every task
  • Any "gotchas" or non-obvious constraints
  • Where to find similar implementations

Keep CLAUDE.md short—under 200 lines. It's consumed on every message, so bloat here affects every interaction.

Common Scenarios

Bug fix: Pass the broken function, the error message, and one level of dependencies. Don't pass the entire file unless the bug requires it.

New feature: Pass interface definitions, one example of a similar feature, and architectural constraints. Let Claude ask for more if needed.

Refactoring: Pass the code to refactor and explain the target state. Include one example of the desired pattern if it exists elsewhere.

Series Recap

What you've learned in Basics:

  1. Context Window — It's working memory, not storage
  2. Token Consumption — History accumulates, output counts too
  3. Accuracy vs Length — Middle gets forgotten, shorter is better
  4. Context Design — Essential vs Helpful vs Noise
  5. Practical Workflow — Scope, minimize, structure, iterate

Key Takeaways

  • Start with a clear, scoped task definition
  • Include only essential context; add more only if needed
  • Structure prompts: task first, code second, constraints at end
  • Set up CLAUDE.md for persistent project context
