Context Design Principles

"I used to dump everything into context. Now I choose carefully."

That mindset shift—from "what can I pass" to "what should I pass"—is the difference between mediocre and excellent results. Context design is about being intentional with every piece of information you include.

The Core Question

Before adding anything to context, ask yourself: "Does Claude NEED this to complete THIS specific task?"

Not "might this be useful someday" or "this seems related." Does Claude need it right now, for this exact task? If the answer isn't a clear yes, leave it out.

The Decision Framework

Categorize everything you might include into three buckets:

Essential

Directly required. Without this, Claude cannot complete the task. Include always.

Helpful

Provides useful context. Might improve accuracy. Include if you have room.

Noise

Unrelated to the task. Will only distract. Never include.

Practical Guidelines

  1. Start minimal, add if needed. Begin with only essential context. If Claude asks questions or produces incomplete results, add more. Don't front-load everything.
  2. Summarize large files. Instead of pasting a 2,000-line file, describe its structure: "This file contains a User model with authentication methods. The relevant function is authenticate() on line 145."
  3. Pass structure, not everything. For bug fixes, pass the broken function and its immediate dependencies—not the entire codebase. For new features, pass the interface definitions and one example of a similar feature.
  4. Separate concerns. Working on authentication? Don't include billing code. Working on the API? Don't include frontend components. Scope tightly.

The goal isn't to give Claude everything it might need. The goal is to give Claude exactly what it needs—no more, no less.

The Checklist

Before sending a prompt, run through this:

  • Is everything I included essential for this task?
  • Is there anything "just in case" that I can remove?
  • Have I put the most important information at the start?
  • Is there a shorter way to convey the same information?

Key Takeaways

  • Ask "what does Claude NEED" not "what CAN I include"
  • Categorize context as Essential, Helpful, or Noise
  • Start minimal, add only if results are incomplete
  • Summarize large files instead of pasting everything
