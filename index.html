The Workflow

Research → Plan → Implement → Review

Research

AI checks docs/ for similar past implementations. Never start from zero.

Plan

AI generates implementation plan. Human reviews and approves before coding.

Implement

Code with the plan as guide. Intent is documented before execution.

Review

AI compares plan vs implementation, detecting drift between intent and code.

Why Now?

Claude Opus-level reasoning makes plans practical

Previous AI assistants could generate text, but not reliable implementation plans. They lacked the ability to investigate codebases thoroughly, understand architectural context, and produce plans worth reviewing.

With Claude Code, the AI can research your codebase, reference past decisions, and generate plans that humans actually want to approve. The bottleneck shifts from "writing documentation" to "reviewing it."

The reason not to document has disappeared.

Real Example

What a plan actually looks like

docs/plans/20250108_auth_refactor.md
# Auth Refactor: JWT to Session-based ## Background Current JWT implementation causes token refresh issues on mobile. Related: docs/plans/20241015_jwt_implementation.md ## Approach 1. Add session store (Redis) 2. Create session middleware 3. Migrate endpoints incrementally 4. Keep JWT for API clients (backward compat) ## Files to Change - src/auth/middleware.ts - Add session check - src/auth/session.ts - New file - src/config/redis.ts - Connection setup ## Risks - Redis single point of failure → Use Redis Cluster - Session fixation → Regenerate ID on login

Plans reference past decisions, document risks, and serve as PR descriptions.
